# Good Code (fateflow-front)

**좋은 코드 = 변경하기 쉬운 코드.** 아래 4기준으로 판단하되 상황에 맞게 유연히.
원리 출처: Toss Frontend Fundamentals. 우리 스택(React 19·TanStack·StyleX/Astryx·FSD v2)에 맞춰 각색. 타입 세부는 [typescript], 폴더 배치는 [fsd], 폼은 [tanstack-form]와 함께 본다.

---

## 1. 가독성 — 한 번에 고려하는 맥락을 줄인다

- **같이 실행되지 않는 코드는 분리.** 서로 다른 조건(role·상태 분기)에서만 도는 코드가 한 컴포넌트에 섞이면, 분기별 컴포넌트로 쪼갠다.
- **구현을 추상화해 상위 의도를 드러내기.** 인증 확인 등 로직을 `<AuthGuard>` 같은 이름으로 감싼다.
- **복잡한 조건식엔 이름.** 여러 줄·중첩 조건은 `isSameCategory`처럼 변수로. 단순식(`arr.map(x => x*2)`)은 그대로.
- **매직 넘버에 이름.** `await delay(300)` → `const ANIMATION_DELAY_MS = 300`. (상수 배치는 [file-naming].)
- **시점 이동 줄이기.** 조건을 이해하려 다른 파일까지 따라가야 하면 호출부에서 바로 보이게 인라인.
- **중첩 삼항 대신 IIFE + if.**
  ```ts
  const status = (() => {
    if (A && B) return "BOTH";
    if (A) return "A";
    if (B) return "B";
    return "NONE";
  })();
  ```
- **범위 비교는 수학 표기 순서로.** `if (80 <= score && score <= 100)`.
- **실행 맥락별로 함수 분리.** 한 훅이 여러 파라미터를 몰아 관리하면 필요한 것만 쓰기 어렵다 → 역할별 전용 훅.

## 2. 예측 가능성 — 이름만 봐도 동작을 알 수 있어야

- **같은 이름 = 같은 동작.** `http.get()`이 몰래 인증 헤더를 붙이면 호출부가 예상 못 함 → 동작이 다르면 이름을 다르게(`getWithAuth`). (우리 `shared/api`는 토큰을 **주입 시임**으로 명시 배선 — 이름 아닌 설정으로 드러냄.)
- **비슷한 역할의 함수는 반환 타입 통일.** 검증 함수가 `boolean` / `{ ok, reason }` 뒤섞이면 위험(object는 항상 truthy). 판별 유니온으로 통일: `type ValidationResult = { ok: true } | { ok: false; reason: string }`. (판별 유니온 세부 → [typescript].)
- **숨은 사이드이펙트 금지.** `fetchBalance`가 로깅까지 하면 예측 불가 → 로깅은 호출 시점에서 명시적으로.

## 3. 응집도 — 함께 바뀌는 코드는 함께 있어야

- **폴더 구조는 우리의 FSD v2가 이 원리의 구현이다.** 타입기반(`components/`·`hooks/`·`utils/`)이 아니라 **도메인(슬라이스) 기반**으로 묶는 것 = 응집도. 단, 구체 배치(layer·slice·segment·배럴)는 **[fsd] 스킬로 위임**한다(우린 `src/domains/`가 아니라 `entities/`·`features/…/{ui,model,api,lib}`).
- **상수는 의미와 함께 한 곳에.** 값이 바뀌면 `ANIMATION_DELAY_MS` 한 곳만.
- **폼 응집도: 필드 단위 vs 폼 단위.** 필드 간 의존 없는 단발 검증은 필드 레벨, 폼 전체가 한 기능이거나 교차 필드면 폼 레벨(Zod 스키마). 우리 폼은 **TanStack Form + Zod** — 세부는 [tanstack-form].

## 4. 결합도 — 수정 시 영향 범위를 작게

- **훅/함수는 하나의 책임만.** 모든 쿼리 파라미터를 한 훅(`usePageState`)에 몰면 하나만 바뀌어도 다 리렌더 → 파라미터별 전용 훅.
- **영향범위가 다른 중복은 분리 유지.** 지금 같아 보여도 앞으로 갈라질 것 같으면 억지로 공통화하지 않는다. 기준: **지금도, 앞으로도 동일하게 동작할 것이 확실할 때만** 공통화.
- **Props Drilling 제거.** 중간 컴포넌트가 쓰지도 않는 props를 통과만 시키면 이름 변경이 연쇄 수정 → ①**조합 패턴(children)** 먼저 ②안 되면 **Context**.

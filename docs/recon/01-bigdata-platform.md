## 4. 데이터 취득 API (익명 JSON) ★핵심 재현법

### 4-1. 시계열 차트 데이터
```
POST https://data.ijto.or.kr/api/dataPick/chart/renderChart.do
Headers: content-type: application/json ; x-requested-with: XMLHttpRequest
         referer: .../sub02/view.do?regSn=N (권장)
Body: {"regSn":"59","chartIndex":0,"searchDataBgnDt":"","searchDataEndDt":""}
```
- **인증 불필요** (JSESSIONID 자동발급). 로그인·회원 없이 조회됨.
- 응답 `data.charts[0].data` = 시계열 배열.
- 공통 필드: `dateVal`(YYYYMM 또는 YYYY), `sumVal`(값), `sumVal_mom`(전월대비%), `sumVal_yoy`(전년동월%), `*_share`(구성비).
- 지표에 따라 **세그먼트 분해 스키마**: 예) 카드소비(regSn=35)는 `내국인 관광객`/`도민`, 연도별 입도(regSn=70)는 `국내`/`국외`/`Total`.
- `searchDataBgnDt`/`searchDataEndDt`로 기간 지정 가능(빈 값이면 기본 최근 ~13개월).
- `chartIndex`로 페이지 내 복수 차트 선택.

### 4-2. 원자료 표 + 메타데이터
- `view.do?regSn=N` 페이지에 **통계표(원시 행 데이터)** 서버 렌더링.
  - 예) regSn=59 → 입항년월/선박명/총승객수/한국인/외국인 승객수.
- 같은 페이지에 **데이터 기본·상세정보**: 기준연도, 관리부서, **갱신주기**, 생성/갱신일시, **데이터 출처**, 데이터 주제, **파일형식(CSV)**, **공개여부/공공누리 유형**.
- 파일형식 CSV 표기 → 다운로드 제공(정확한 다운로드 URL은 적재 단계에서 지표별 확인).

## 7. 활용 각도 (이데이션 참조)
- **수요 예측·시즌성**: 크루즈/항공/입도 시계열의 MoM·YoY로 성수기/비수기, 전년비 회복세 파악.
- **타깃 정의**: 거주지 분포(67), 방문형태(40), 국가별 외국인 소비(60)로 세그먼트.
- **동선·체류**: 지역별 도착(51/48), 이동통신 지역방문(21), 섬속섬(57)로 공간 수요.
- **만족도·페인포인트**: 실태조사(85/87/88)로 크루즈·외국인 만족 세부 항목(출입국 절차 등).
- **소비 설계**: 카드소비(35)·음식점업(82)·외식물가(트렌드리포트)로 지출 구조.

## 8. 주의/위험
- renderChart.do는 same-origin 전제(referer/x-requested-with). 외부 재현 시 헤더·세션 필요할 수 있음 → 브라우저 컨텍스트(또는 세션 취득 후) 호출 권장.
- 지표별 응답 스키마가 달라(세그먼트) 일괄 파싱 시 필드 동적 처리 필요.
- 최신월은 '잠정' 값일 수 있음(일일통계 기준).
